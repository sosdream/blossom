    UCOS-II源码分析

    System ucos real-time system
    1. $\mu$C/OS-$\amalg$初始化
    2. $\mu$C/OS-$\amalg$的启动

    $\mu$C/OS-$\amalg$是一款典型的实时操作系统，通过对它的分析可以很好地掌握实时操作系统设计中所需要注意的细节。因下载$\mu$C/OS-$\amalg$需要翻墙且访问很慢，这里在github上做了个$\mu$C/OS-$\amalg$的源码仓库提供下载。

    [$\mu$C/OS-$\amalg$源码][]

    学习操作系统一般遵循如下步骤进行分解学习。

    1. 启动和初始化
    2. 中断处理
    3. 内存管理
    4. 任务管理
    5. 进程间通信

    当然对于如Linux这样的操作系统来说，设备管理及其子系统也是非常重要且不可或缺的，但实时操作系统最重要而且必不可少的还是以上几个方面。我们先从$\mu$C/OS-$\amalg$的启动及其初始化开始逐步深入。

    如下为$\mu$C/OS-$\amalg$的源码文件结构。

    $\mu$C/OS-$\amalg$初始化

    $\mu$C/OS-$\amalg$初始化的关键函数是OSInit(),在这个阶段，会创建一个Idle任务和一个Stat任务(如果配置了的话)，其中Idle任务优先级是最低的，它会在没有任务运行时运行，直到有新的任务需要运行。下图展示了OSInit()对相关变量和结构的初始化。

    从上图可知，任务控制快TCB是链接在一个双向链表中的，OSTCBList是这个双向链表的头指针。当有新的任务被创建时，新的任务的TCB总时放入这个链表的头部，也就是说OSTCBList总时指向最近新加入系统的任务。

    OSInit()除了初始化并创建了上面两个任务外，为了提高内存分配效率，$\mu$C/OS-$\amalg$中还实现了5种类型的内存池，分别为OS_TCB、OS_EVENT、OS_Q、OS_FLAG_GRP、OS_MEM,其实现方式是通过5个单向链表链接这几种$\mu$C/OS-$\amalg$中常用数据结构的空闲内存块来实现的，同时对于每个空闲链表所包含空闲内存块的数量可在OS_CFG.H文件中进行配置。如下图所示。

    $\mu$C/OS-$\amalg$的启动

    $\mu$C/OS-$\amalg$在调用OSInit()初始化完成后，再创建一个用户自己的任务就可以启动运行了，启动$\mu$C/OS-$\amalg$并让它开始调度运行是由OSStart()函数完成的。
